Setup Firebase

iLoveConverts Pro using the firebase database for changing the application logos, titles etc.

  • First, If you don't have any firebase account then you need to create a firebase account.
  • You will need to start by setting up your project on the Firebase console. First, go here to find the console.
  • Click on the card with the plus sign that says Add project. In the pop-up that appears you can edit the project name and settings. Change the name to anything you like, accept the terms and conditions, and leave everything else at its default.
  • In the Firebase console click on the gear icon next to Project Overview to open the project settings. In the General tab scroll down to the bottom. You will see a notice telling you that there are no apps in your project. You will not be adding an app here but you can get the information by clicking on the web icon “”. A pop-up appears with your API key, database URL etc. Copy the information of `firebaseConfig`.
  • Next, Open your `iLoveConvertsPro` and go into the project `src/environments/environment.ts` & `src/environments/environment.prod.ts` file and paste here firebase configuration information. In the same way, open `AdminPanel` and go into the project `src/environments/environment.ts` & `src/environments/environment.prod.ts` files and paste this information.
  • Create a admin user for admin panel login. In the firebase console go into the Authentication section. In the `Users` tab click on Add User button and fill up your admin panel email and password which is necessary for admin panel login.
  • And next go into the `Sign-in method` tab and click on Email/Password from the list and enable and save it.
  • Goo into the Cloud Firestore section and click on Rules tab and replace the rules with the following rules.
    • service cloud.firestore {
    match /databases/{database}/documents {
        match /{document=**} {
            allow read;
            allow write: if request.auth != null;
        }
    }
}
  • Finally go into the Storage section and click on Rules tab and replace the rules with the following rules.
    • rules_version = '2';
service firebase.storage {
    match /b/{bucket}/o {
        match /{allPaths=**} {
            allow read;
            allow write: if request.auth != null;
        }
    }
}

Configuration of Meta Tags

A quick rundown of the Angular Meta service, which makes it easy to get and set various meta tags on a route

We have created a service for meta tags in our project. You will find that service at - `src/app/services/meta-service.service.ts`. At the top of this service we have declare two variables - websiteTitle & websiteUrl, you should change these two variables with your website title and website Url. 

// Change and replace with your website information
websiteTitle: any = 'your website title';
websiteUrl: any = 'your website url';
})
At this service you will get five functions for adding meta tags in your website. Functions are -
  • setTitle() - Set Meta Title of your sites
  • addMetaTags() - Add Meta Tags

    • -- Retrieves or creates a set of meta tag elements in the current HTML document. In this function you can modify and replace content of the meta tags. 

    // You can change and update the `content` of meta value
addMetaTags() {
    this.meta.addTags([
    { name: 'author', content: 'Add Author name/Website Name' },
    { name: 'description', content: 'Add short description of current document' },
    { name: 'keywords', content: 'Add Some keywords of current document' },
    
    // Open Graph / Facebook
    { property: "og:type", content: 'Your website URL' },
    { property: "og:url", content: 'Your website URL' },
    {property: 'og:title', content: 'Add Some keywords of current document' },
    {property: "og:description", content: 'Add short description of current document' },
    
    // Twitter
    { property: "twitter:url", content: 'Your website URL' },
    { property: "twitter:title", content: 'Add Some keywords of current document' },
    { property: "twitter:description", content: 'Add short description of current document' }
]);
}
  • getMetaTags() - Get Meta Tags

    • -- This meta service function retrieves meta tag element in the current HTML document.

  • updateMetaTags() - Update Meta Tags

    • -- This function helps to modifies an existing meta tag element in the current HTML document. This function is called to change meta tags from different modules.

    // Update Meta Tags
// @Params {obj} Dynamic content of meta values
// By calling this function It will immediately update the meta keywords for specific routes 
updateMetaTags(obj) {
    this.meta.updateTag({ name: 'description', content: obj.description });
    this.meta.updateTag({ name: 'keywords', content: obj.keywords });
    // Open Graph / Facebook
    this.meta.updateTag({ property: 'og:url', content: obj.ogUrl });
    this.meta.updateTag({ property: 'og:title', content: obj.ogtitle });
    this.meta.updateTag({ property: 'og:description', content: obj.ogDescription });
    // Twitter
    this.meta.updateTag({ property: "twitter:url", content: obj.twitterUrl });
    this.meta.updateTag({ property: "twitter:title", content: obj.twitterTitle });
    this.meta.updateTag({ property: "twitter:description", content: obj.twitterDescription });
}

    We have called this function from various modules to update meta tags. You can update meta tags from these modules.

    1. About Page: Update meta tags on about page from src/app/pages/about/about.component.ts
    2. Conversion Tools Page: Update meta tags on conversion tools page from src/app/pages/conversion-tools/conversion-tools.component.ts
    3. Encryption Tools Page: Update meta tags on Encryption Tools page from src/app/pages/encryption-tools-converter/encryption-tools/encryption-tools.component.ts
    4. Image Compress: Update meta tags on Image Compressout page from src/app/pages/image/image-compress/image-compress-upload/image-compress-upload.component.ts
    5. Image Converter Page: Update meta tags on Image Converter page from src/app/pages/image/image-converter/image-upload/image-upload.component.ts
    6. Number Tools Converter Page: Update meta tags on Number Tools Converter page from src/app/pages/number-tools-converter/number-tools/number-tools.component.ts
    7. OCR Tools: Update meta tags on OCR page from src/app/pages/ocr/ocr.component.ts
    8. Privacy Policy: Update meta tags on Privacy Policy page from src/app/pages/privacy-policy/privacy-policy.component.ts
    9. Term & Conditions: Update meta tags on Term & Conditions page from src/app/pages/terms/terms.component.ts
    10. Text Tools: Update meta tags on Text Tools page from src/app/pages/text-tools-converter/text-tools/text-tools.component.ts
    11. Unit Converter Tools: Update meta tags on Unit Converter Tools page from src/app/pages/unit-converter/all-unit-converter/all-unit-converter.component.ts & src/app/pages/unit-converter/each-unit-converter/each-unit-converter.component.ts

  • removeMetaTags() - Remove Meta Tags

    • -- This function helps to remove an existing meta tag element from the current HTML document.

    Production Build/Deployment

    You can get a preview of your application using the ng serve --prod command that starts a local HTTP server such that the application with production files is accessible using - http://localhost:4200.

    ng serve --prod

    During development, we use the ng serve command to build, watch, and serve the application from local memory. For a production usage, you have to deploy all the files from the dist folder in the HTTP server of your choice.

    The output folder is dist/project-name/ by default. For example this project name is iLoveConvertsPro so the output folder you will find at dist/iLoveConvertsPro

    When you are ready to deploy, however, you must use the ng build command to build the app and deploy the build artifacts elsewhere.

    ng build --prod

    Overview of basic deployment to remote server

    For the simplest deployment, create a production build and copy the output directory to a web server.

    • Start with the production build:

      • ng build --prod

    • Copy everything within the output folder (dist/project-name/ by default) to a folder on the server.

    For details documentation - https://angular.io/guide/deployment
